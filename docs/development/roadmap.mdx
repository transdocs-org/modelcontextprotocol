## 智能代理（Agents）

随着 MCP 越来越深入地融入代理工作流，我们正专注于以下关键改进：

- **异步操作**：支持可能持续较长时间的长时间运行操作，并具备断开连接和重新连接的弹性处理能力

## 认证与安全

我们正在改进授权和安全机制，以提升用户安全性并优化开发者体验：

- **指南与最佳实践**：通过指南和最佳实践文档，帮助开发者避免常见错误，安全地部署 MCP
- **动态客户端注册（DCR）的替代方案**：探索 DCR 的替代方案，在保持良好用户体验的同时解决运维方面的挑战
- **细粒度授权**：为敏感操作开发基础授权机制和指南
- **企业托管授权**：支持企业通过单点登录（SSO）简化 MCP 服务器的授权管理
- **安全授权流程集成**：使开发者能够集成安全的授权流程，用于主 MCP 服务器授权之外的下游 API

## 验证

为了打造一个稳健的开发者生态系统，我们计划在以下方面投入资源：

- **参考客户端实现**：通过高质量的 AI 应用展示协议的各项功能
- **参考服务端实现**：展示认证模式和远程部署的最佳实践
- **合规性测试套件**：自动化验证客户端、服务端和 SDK 是否正确实现了规范

这些工具将帮助开发者自信地实现 MCP，同时确保整个生态系统的行为一致性。

## 注册中心（Registry）

为了充分发挥 MCP 的潜力，我们需要更便捷的方式来分发和发现 MCP 服务端。

我们计划开发一个 [**MCP 注册中心**](https://github.com/orgs/modelcontextprotocol/discussions/159)，以实现服务端的集中发现和元数据管理。该注册中心将主要作为一个 API 层，供第三方市场和发现服务在此基础上构建。

## 多模态（Multimodality）

在 MCP 中全面支持 AI 的多模态能力，包括：

- **新增模态类型**：视频及其他媒体类型
- **[流式传输](https://github.com/modelcontextprotocol/specification/issues/117)**：支持多部分、分块消息以及用于交互式体验的双向通信
