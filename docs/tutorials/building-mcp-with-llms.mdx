本指南将帮助你使用LLM来构建自定义的模型上下文协议（MCP）服务器和客户端。本教程将以Claude为例进行讲解，但你也可以使用其他前沿的LLM完成类似任务。

## 准备文档

开始之前，请收集必要的文档，以帮助Claude理解MCP：

1. 访问 https://modelcontextprotocol.io/llms-full.txt 并复制完整文档文本
2. 前往 [MCP TypeScript SDK](https://github.com/modelcontextprotocol/typescript-sdk) 或 [Python SDK仓库](https://github.com/modelcontextprotocol/python-sdk)
3. 复制README文件及其他相关文档
4. 将这些文档粘贴到你与Claude的对话中

## 描述你的服务器

提供文档后，向Claude清楚地描述你想要构建的服务器类型。请具体说明以下内容：

- 你的服务器将暴露哪些资源
- 它将提供哪些工具
- 需要包含哪些提示词（prompt）
- 需要与哪些外部系统进行交互

例如：

```
构建一个MCP服务器，要求：
- 连接我公司的PostgreSQL数据库
- 将表结构作为资源暴露
- 提供执行只读SQL查询的工具
- 包含常见数据分析任务的提示词
```

## 与Claude协作

在使用Claude构建MCP服务器时：

1. 先从核心功能开始，逐步迭代增加更多功能
2. 对于不理解的代码部分，向Claude询问解释
3. 根据需要请求修改或改进
4. 让Claude帮助你测试服务器并处理边界情况

Claude可以帮助实现所有关键的MCP功能：

- 资源管理与暴露
- 工具定义与实现
- 提示词模板与处理程序
- 错误处理与日志记录
- 连接和传输设置

## 最佳实践

在使用Claude构建MCP服务器时：

- 将复杂的服务器分解为更小的部分
- 在继续下一步前，彻底测试每个组件
- 注意安全性 - 验证输入并适当限制访问
- 为代码编写良好的文档以便未来维护
- 仔细遵循MCP协议规范

## 后续步骤

在Claude帮助你构建完服务器后：

1. 仔细审查生成的代码
2. 使用MCP Inspector工具测试服务器
3. 将其连接到Claude.app或其他MCP客户端
4. 根据实际使用情况和反馈进行迭代改进

请记住，当需求随时间变化时，Claude可以帮助你修改和改进服务器。

需要更多指导？只需向Claude提出关于实现MCP功能或解决出现问题的具体问题即可。
